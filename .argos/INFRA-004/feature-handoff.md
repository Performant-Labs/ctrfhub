## Key Design Decisions

1. **Better Auth tables excluded from migrations** via `schemaGenerator.skipTables`. Organization and User entities exist solely for ORM relationship mapping. CTRFHub never creates/alters those tables.

2. **String types for enum-like fields** (status, artifactType, storageType, aiCategory). ENUMs aren't portable between PG and SQLite — the skill doc (`mikroorm-dual-dialect.md`) bans them. Validation happens at the Zod/service layer.

3. **Integer PKs for CTRFHub tables, string PKs for Better Auth tables.** Better Auth uses string UUIDs; our tables use auto-increment integers per the database-design.md spec.

4. **`p.string().length(N)`** — MikroORM v7.0.12 uses the chainable `.length()` method, not a constructor argument. The skill doc's `p.string(100)` example is incorrect for this version.

5. **`.gitkeep.ts` → `.gitkeep`** — The TS placeholder files were being loaded by MikroORM's migration runner as migration classes, causing `MigrationClass is not a constructor` errors. Plain `.gitkeep` files avoid this.
