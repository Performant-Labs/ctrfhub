## Decisions not covered by planning docs

- **Zod v4 instead of v3:** `@fastify/type-provider-zod@1.0.0` requires `zod@>=4.2.0` as a peer dependency. Zod v4 is now the latest release (4.3.6). The planning docs reference Zod v3 API examples (`z.infer<>`, `z.object()`, etc.), which remain compatible in Zod v4 via the `zod/v3` compat shim if needed, but the v4 native API is the same for our usage. The `zod-schema-first.md` skill patterns work unchanged. This adjoins `docs/planning/architecture.md §Backend (Zod row)`.
- **`@fastify/swagger` + `openapi-types` added as dependencies:** Required as peer deps of `@fastify/type-provider-zod@1.0.0`. Not mentioned in the architecture doc but implied by the type-provider choice. Adjoins `architecture.md §Backend`.
- **`playwright-ctrf-json-reporter` (unscoped) instead of `@ctrf-io/playwright-ctrf-json-reporter`:** The testing-strategy.md references the `@ctrf-io/` scoped name, but the actual npm package is published as `playwright-ctrf-json-reporter` (unscoped). The scoped version returns 404 on npm. Adjoins `docs/planning/testing-strategy.md §E2E Reporter config`.
- **`@fastify/static@^9`, `@fastify/view@^11`:** Planning docs don't pin versions; speculative pinning to older majors caused install failures. Corrected to match actual npm latest. Adjoins `architecture.md §Backend`.
- **ESLint v8 (deprecated) used:** The `@typescript-eslint/parser@8.x` still supports ESLint v8 but ESLint v8 shows a deprecation warning. Flat config migration (ESLint v9+) can be done in a follow-up but is out of scope for INFRA-001. Adjoins `docs/planning/tasks.md §INFRA-001` (acceptance says "ESLint config in place").
- **Node.js v25.9.0 local runtime:** The user's machine runs Node 25 (Current), not Node 22 LTS as specified in `architecture.md`. The code targets Node 22 LTS via `engines` field and ES2022 target — it will work on both. The CI pipeline (CI-001) should pin Node 22. No code change needed.
